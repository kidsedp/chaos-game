The Chaos Game

Try adjusting the settings and hitting "restart" when you've found some you'd like to try out!

What is the Chaos Game?

The Chaos Game (wikipedia) is a method for creating ordered images out of randomness. It follows a simple series of instructions, which can be summarized as follows:

  1. Pick n corner points at random.
  2. Pick a starting point p at random.
  3. Each turn, randomly choose one of the corner points c and move some percentage of the way d toward that point. (Let's say halfway.) Let p be equal to this new location.
  4. Draw a dot at p.
  5. Go back to step 3; repeat forever.

What do the settings do?

  • Points per frame
    The number of points that are drawn on the screen after each frame. Basically, how fast the image is drawn.
  • Distance from last point to next corner
    Where the next point will be chosen. This is d in step 3 of the algorithm. This seems to control how "spaced out" the elements in the image appear.
  • Pixel size
    How big each dot is. Bigger pixels are easier to follow, but smaller pixels look nicer.
  • Number of corners
    The number of initial points to draw. This is n in step 1 of the algorithm.
  • How far the next corner can be from the last
    Normally, any corner in the Chaos Game is allowed to be chosen each turn. This settings limits the corners that can be chosen to those c distance away from the previous point that was chosen. For example, given a square (4 corners), and a setting of 1 on this slider, the corners adjacent to the previous point are eligible to be chosen, but not the point across from the previous point. Note that values higher than half of the number of points (e.g. ≥2 for triangles, ≥3 for quadrilaterals or pentagons) will have no effect.
  • Fixed corners/random corners
    Fixed corners ensure the corners form a regular polygon (equal spacing between all points). This ensures that you can see the image that's created as clearly as possible.
  • Disallow repeats/allow repeats
    The traditional version of the Chaos Game allow any corner to be chosen in step 3 of the algorithm. However, adding the condition that a corner cannot be chosen twice in a row can create some interesting patterns. Disallowing repeats enforces this rule.
